03 PRODUCT BREAKTHROUGH METHODOLOGY D0 产品突破方法论

D0 的产品设计遵循五条核心原则。每一条都指向同一个目标：让 AI Agent 成为一等公民。

01

CLI-as-API / 命令行即接口

不是先做 GUI 再包 API，而是 CLI 本身就是标准接口。AI Agent 可以像人类一样直接调用命令行，无需额外 SDK 或封装层。这消除了传统 API 集成的摩擦。

02

Zero-Config Progressive Disclosure / 零配置渐进暴露

读取操作（价格、行情、账户）零配置直接可用，降低首次体验门槛到接近零。只有执行交易时才需要配置私钥。80% 的功能不需要任何 setup。

03

Non-Custodial by Design / 架构级非托管

不是 "我们承诺不动你的钱"，而是架构上不可能。私钥永远在本地，EIP-712 签名在用户设备上完成，D0 从不接触私钥。信任不来自承诺，来自密码学。

04

Exchange Abstraction / 交易所抽象层

Hyperliquid、Polymarket、以及未来的更多协议，通过统一的命令语义对 Agent 暴露。Agent 不需要知道底层是哪条链、哪个协议，D0 处理所有差异。

05

Composable Skill System / 可组合技能系统

D0 作为 Clawhub 上的 AI Skill 发布，可以与其他 Skill 自由组合。一个 Agent 可以同时使用 D0 (交易) + 数据分析 Skill + 新闻 Skill，形成完整的自主交易 pipeline。

05 DONUT RESEARCH 研究方向

Donut Research 聚焦 AI 交易基础设施的核心问题。以下是当前活跃研究课题。

ACTIVE DR-001

AI Agent 交易意图解析的准确率边界

研究 LLM 在模糊交易指令下的意图识别准确率。"帮我搞点 ETH" vs "市价买入 0.5 ETH" — 模糊度越高，误执行风险越大。如何设计 confirmation loop 在自主性和安全性之间取得平衡？

INTENT PARSING SAFETY CONFIRMATION UX
ACTIVE DR-002

跨协议统一执行语义

Hyperliquid (永续合约) 和 Polymarket (预测市场) 的订单语义完全不同。如何定义一套统一的抽象层，让 Agent 无需感知底层差异，同时不丢失协议特有功能？

ABSTRACTION MULTI-PROTOCOL SEMANTIC DESIGN
EXPLORING DR-003

Agent 自主风控的失效模式分析

当市场极端波动时 (黑天鹅)，Agent 的风控逻辑可能因数据延迟、流动性枯竭或级联清算而失效。研究 failsafe 机制：硬性止损 vs 熔断 vs 人工接管触发条件。

RISK CONTROL BLACK SWAN FAILSAFE
EXPLORING DR-004

多 Agent 协同交易中的博弈均衡

当多个 AI Agent 同时使用相同策略 (如资金费率套利) 时，alpha 会快速消失。研究 Agent 间的博弈动态：信息不对称如何被 AI 压缩？策略同质化的均衡点在哪？

GAME THEORY ALPHA DECAY MULTI-AGENT
EXPLORING DR-005

链上行为指纹与 MM 模式识别

做市商 (MM) 的链上行为有可识别的模式 — 阶梯拉盘 (C型)、掩护出货 (E型)。AI 能否从链上数据实时检测 MM 行为模式？准确率和延迟的 trade-off 在哪？

ON-CHAIN ANALYTICS PATTERN RECOGNITION MM DETECTION
