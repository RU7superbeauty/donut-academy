## 三种策略对应三种市场状态

量化策略没有万能公式，关键是**匹配市场状态**：

```
震荡市（无明显趋势）→ 网格交易
趋势市（单边上涨/下跌）→ 趋势跟踪
回归市（均值回归倾向）→ 均值回归/配对交易
```

第一步永远是判断当前市场状态：

```bash
d0 ta ETH
# 看 Signal 和 MA 排列判断市场状态
```

---

## 策略一：网格交易

在价格区间内均匀分布买卖单，价格震荡时持续获利：

```
价格区间：$3,000 - $3,500
网格间距：$100（6个网格）

$3,500 → 卖出
$3,400 → 卖出
$3,300 → 卖出
$3,200 → 当前价格
$3,100 → 买入
$3,000 → 买入

每次价格穿越一个网格 = 一笔利润
```

**D0 实现**：

```bash
# 设置网格（手动版，每个价位下单）
d0 hl:limit buy ETH 0.05 3100
d0 hl:limit buy ETH 0.05 3000
d0 hl:limit sell ETH 0.05 3300
d0 hl:limit sell ETH 0.05 3400

# 查看所有挂单
d0 hl:orders
```

**适用条件**：
- ETH 在确认的区间内横盘
- `d0 ta` 显示 RSI 40-60，无明显趋势
- 预期震荡持续 > 3 天

**风险**：价格单边突破区间，导致仓位严重偏向一侧。必须设置整体止损。

## 策略二：趋势跟踪

顺势而为，用均线交叉或动量信号确认趋势后入场：

```
信号：MA20 上穿 MA50（金叉）
入场：金叉后第一根收阳 K 线的收盘价附近
止损：最近支撑位下方
目标：跟随趋势，分批止盈
```

**D0 实现**：

```bash
# 1. 确认趋势信号
d0 ta ETH
# 期望：MACD bullish crossover，MA20 > MA50，RSI 50-65

# 2. 建仓（趋势方向）
d0 hl:leverage ETH 2
d0 hl:limit buy ETH 0.1 3200

# 3. 止损在最近支撑
d0 hl:stop-loss ETH 0.1 3040

# 4. 持仓期间定期检查
d0 hl:positions
d0 ta ETH  # 监控趋势是否逆转
```

**持仓原则**：趋势信号反转（MACD 死叉，价格跌破 MA20）才考虑平仓，不要过早止盈。

## 策略三：均值回归

当价格大幅偏离历史均值时，押注其回归：

```
逻辑：ETH 的 BTC 比值（ETH/BTC）长期稳定在某个范围
当比值过高 → 做空 ETH，做多 BTC（或只做空 ETH）
当比值过低 → 做多 ETH，做空 BTC（或只做多 ETH）
```

**D0 实现（单资产版）**：

```bash
# 当 ETH RSI 极度超卖（< 25）且偏离 MA50 超过 15%
d0 ta ETH
# RSI: 22.1, Price: $2,720, MA50: $3,200 (偏离 -15%)

# 建仓（均值回归，小仓位）
d0 hl:limit buy ETH 0.05 2720
d0 hl:stop-loss ETH 0.05 2500  # 止损在 -8%

# 目标：回归 MA50 = $3,200，约 +17%
```

## 策略选择决策树

```
当前 d0 ta 显示：

RSI 40-60 + 无明显趋势
    → 网格交易

RSI 方向性 + MACD 金叉/死叉
    → 趋势跟踪（顺势方向）

RSI 极值（< 25 或 > 75）+ 价格大幅偏离均线
    → 均值回归（反向入场）
```

## 核心概念回顾

- **市场状态识别**：先判断震荡/趋势/极端偏离，再选策略
- **网格**：震荡市的现金机器，突破时有风险
- **趋势跟踪**："让利润奔跑"，信号确认再入场
- **均值回归**：押注极端回归正常，小仓位，止损要严格
